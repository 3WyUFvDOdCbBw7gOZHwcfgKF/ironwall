# Ironwall 概览

Ironwall 是一门强类型、安全优先的语言。它的设计偏向清楚规则，而不是语言复杂度；偏向可审计的运行时代价，而不是隐藏机制；偏向直接失败，而不是用恢复机制掩盖已经破坏的不变式。

Ironwall 保持核心语言模型紧凑、可检查。它只接受一组有限的抽象能力，用来提升实际表达力，而不是把语言变成难以审计的系统。

## 关键特性

- 编译到 native 目标
- 强类型，显式类型优先
- 主要采用名义类型
- 高阶函数
- 带穷尽 `match` 的 union type
- 显式实例化的简单泛型
- 强制垃圾回收
- 显式触发的 Stop-The-World GC
- 没有宏
- 没有 `null`
- 没有 type alias
- 没有继承
- 没有语言级异常

## 设计概要

Ironwall 有意保持简单的语言表面。绑定、函数参数、返回类型、属性、全局变量和泛型实例化都必须显式书写。类形成名义类型，泛型实例仍然是名义类型，赋值兼容规则保持保守。

可恢复的不确定性应使用数据建模，尤其是 `union` 和 `match`，而不是使用 `null` 或异常。非法 tag、前提条件失败、数组越界等不可恢复违规会立即失败。

运行时设计同样强调显式。GC 是内存安全模型的强制部分，但回收不会隐藏在后台启发式策略之后。Ironwall 采用简单的 Stop-The-World Mark-Sweep 方向，回收必须由程序显式触发。
