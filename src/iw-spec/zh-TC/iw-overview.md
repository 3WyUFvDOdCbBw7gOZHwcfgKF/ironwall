# Ironwall 概覽

Ironwall 是一門強型別、安全優先的語言，目標是提供顯式邊界、可預測的執行期行為，以及較低的實作複雜度。它的設計偏向清楚規則，而不是語言魔法；偏向可審計的執行期代價，而不是隱藏機制；偏向直接失敗，而不是用恢復機制掩蓋已經破壞的不變式。

Ironwall 以編譯到 native 目標為方向，同時保持核心語言模型緊湊、可檢查。它只接受一組有限的抽象能力，用來提升實際表達力，而不是把語言變成難以審計的系統。

## 關鍵特性

- 編譯到 native 目標
- 強型別，顯式型別優先
- 主要採用名義型別
- 高階函數
- 帶窮盡 `match` 的 union type
- 顯式實例化的簡單泛型
- 強制垃圾回收
- 顯式觸發的 Stop-The-World GC
- 沒有宏
- 沒有 `null`
- 沒有 type alias
- 沒有繼承
- 沒有語言級異常

## 設計概要

Ironwall 有意保持儘可能簡單的語言表面。綁定、函數參數、返回型別、屬性、全域變數和泛型實例化都必須顯式書寫。類形成名義型別，泛型實例仍然是名義型別，賦值相容規則保持保守。

可恢復的不確定性應使用資料建模，尤其是 `union` 和 `match`，而不是使用 `null` 或異常。非法 tag、前提條件失敗、陣列越界等不可恢復違規會立即失敗。

執行期設計同樣強調顯式。GC 是記憶體安全模型的強制部分，但回收不會隱藏在背景啟發式策略之後。Ironwall 採用簡單的 Stop-The-World Mark-Sweep 方向，回收必須由程式或 host interface 顯式觸發。
